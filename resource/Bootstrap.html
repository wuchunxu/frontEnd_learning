    响应式

    如何编写响应式网站

    声明viewport(视口)元标签

    <meta name="viewport" content="width=device-width,initial-scale=1">

    早期3G手机为浏览器大尺寸网页，只能强行页面缩小,导致图片/文字/链接很小。IOS提出“视口”概念，视口用来盛放网页内容，尺寸可以指定，超过部分可以滑动网页来查看。

    width=device-width宽度等于设备宽度

    initial-scale=1初始化缩放比例为1

    使用流式布局

    float:left;/display:inline-block;

    使用相对尺寸

    容器、文字、图片都使用相对尺寸

    .container{
    width:50%;
}
body{
    font-size:2em;
}
img{
    width:25%;
}

    【重点】媒体查询技术(CSS3--Media Query)

    Media:指浏览网页的设备，如screen(pc/pad/phone),print,tv。

    Media Query:查询当前浏览网页设备类型、特性(解析度、对比、尺寸、手持方向等),而选择性执行某些css代码，而忽略另外一些。

    使用媒体查询有两种方法:

    (1)根据媒体查询的结果执行不同的CSS文件；
    <link  media="screen and (min-width:768px) and (max-width:991px)" rel="stylesheet" href="1.css"/>
<link  media="print" href="2.css"/>
    (2)根据媒体查询结果执行CSS片段汇总不同的部分。
    /*pad*/
@media screen and (min-width:768px) and (max-width:991px){
    //css code
}
    PC >=992px
    pad 768~991px
    phone <=767px

    栅格系统

    最外层必须套上容器

    div.container/container-fluid

    容器中可以放置任何内容。但，若想使用栅格系统，必须使用.row

    div.container>.div.row

    .row中只能放置.col，.col中可以放置任何内容。

    每行默认分为12等分，每个列必须指定占比

    栅格系统针对不同的屏幕提供了不同的class，.col-lg-1,.col-md-1,.col-sm-1,.col-xs-1

    可以使用“列偏移”实现指定列及其后的列向右偏移的效果。.col-md-offset-2

    注意不同屏幕下列的适用性问题

    .col-xs-* 适用于xs/sm/md/lg屏幕
    .col-sm-* 适用于sm/md/lg屏幕
    .col-md-* 适用于md/lg屏幕
    .col-lg-* 适用于lg屏幕

    一个列可以指定不同屏幕下的宽度占比

    div.col-xs-12.col-sm-9.col-md-6

    常见的错误：div.col-xs-6.col-md-6，前者已经声明了后者的效果。

    一个列可以指定特定的屏幕隐藏

    .hidden-xs 仅在xs屏幕下隐藏
    .hidden-sm 仅在sm屏幕下隐藏
    .hidden-md 仅在md屏幕下隐藏
    .hidden-lg 仅在lg 屏幕下隐藏
    .hidden

    从农耕到工业

    学习基础知识是我们进入某个领域的前提，但并不意味着，我们要纯靠基础知识来实践。明明是工业化的时代，却用锄头打造自己的文明，显然是与时代的发展相违背的。

    如何做好一件事情

    有一次我在“得到”app上听到清华大学吴军老师的分享，要想做好一件事情，有3个要素：基线baseline、目标target和路径path。三者组成了一个Z字型。如果要在一个很专业的领域发展，首先要做的事情是提高自己的基线。在高速发展的今天，“民科”很难有所成就，因为，很多领域的基线非常高，如果还致力于研究200百多年前的知识，显然不妥。
